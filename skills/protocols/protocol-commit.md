# protocol-commit

> Источник: `.cursorrules` (протокол по командам "комить", "закомить" и т.п.)

## Базовый сценарий

1. Выполнить `git status`, проверить staging.
2. Определить тему изменений (MBB vs Cursor) и разделить коммиты при необходимости.
3. Сформировать краткое и точное имя коммита.
4. `git add` нужных файлов → `git commit -m`.
5. Проверить `git status`.

## Ограничения

- Коммит только по явной команде пользователя.
- Не смешивать изменения настроек и разработки, если оба типа присутствуют.
